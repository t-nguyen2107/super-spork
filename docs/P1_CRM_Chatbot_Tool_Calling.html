CRM Chatbot
Tool Calling

A chatbot that answers natural language questions about your CRM contacts and deals by calling structured, parameterised database tools — no raw SQL generation, no schema exposure.

Ollama · local CPU PostgreSQL · Docker qwen2.5:7b Python

Step 1 of 6

Pull the Ollama Model

For a CPU-only machine, qwen2.5:7b is the recommended model — native tool/function-calling support, fits in ~8 GB RAM, runs at ~8–15 tokens/sec on a modern i7/Ryzen.

bash
# Check what's already pulled
ollama list

# Pull the recommended model (one-time, ~4.7 GB download)
ollama pull qwen2.5:7b

# Verify it loaded correctly
ollama run qwen2.5:7b "say hello"
# Exit with /bye

3b. Schema

schema.sql
-- contacts: 78k in production, ~50 in local seed
CREATE TABLE contacts (
    id            SERIAL PRIMARY KEY,
    name          TEXT NOT NULL,
    email         TEXT UNIQUE NOT NULL,
    company       TEXT,
    phone         TEXT,
    city          TEXT,
    country       TEXT,
    industry      TEXT,  -- insurance | fintech | saas | manufacturing | retail
    status        TEXT NOT NULL DEFAULT 'active'
                  CHECK (status IN ('active','inactive','lead','churned')),
    source        TEXT,
    assigned_to   TEXT,
    tags          TEXT[],  -- e.g. ARRAY['vip','renewal-risk']
    notes         TEXT,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE deals (
    id            SERIAL PRIMARY KEY,
    contact_id    INT NOT NULL REFERENCES contacts(id) ON DELETE CASCADE,
    title         TEXT NOT NULL,
    value         NUMERIC(14,2) NOT NULL CHECK (value >= 0),
    currency      TEXT NOT NULL DEFAULT 'USD',
    stage         TEXT NOT NULL DEFAULT 'prospecting'
                  CHECK (stage IN (
                    'prospecting','qualified','proposal',
                    'negotiation','closed_won','closed_lost')),
    probability   INT DEFAULT 10 CHECK (probability BETWEEN 0 AND 100),
    owner         TEXT,
    close_date    DATE,
    product       TEXT,
    notes         TEXT,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE activities (
    id            SERIAL PRIMARY KEY,
    contact_id    INT REFERENCES contacts(id) ON DELETE CASCADE,
    deal_id       INT REFERENCES deals(id) ON DELETE SET NULL,
    type          TEXT CHECK (type IN ('call','email','meeting','note','demo')),
    summary       TEXT,
    created_by    TEXT,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Indexes
CREATE INDEX idx_contacts_status    ON contacts(status);
CREATE INDEX idx_contacts_company   ON contacts(company);
CREATE INDEX idx_contacts_country   ON contacts(country);
CREATE INDEX idx_contacts_industry  ON contacts(industry);
CREATE INDEX idx_contacts_name_trgm ON contacts USING gin(name gin_trgm_ops);
CREATE INDEX idx_deals_stage        ON deals(stage);
CREATE INDEX idx_deals_owner        ON deals(owner);
CREATE INDEX idx_deals_value        ON deals(value DESC);
CREATE INDEX idx_deals_close_date   ON deals(close_date);

-- Auto-update updated_at
CREATE OR REPLACE FUNCTION touch_updated_at()
RETURNS TRIGGER LANGUAGE plpgsql AS $$
BEGIN NEW.updated_at = now(); RETURN NEW; END;
$$;
CREATE TRIGGER trg_contacts_updated BEFORE UPDATE ON contacts
  FOR EACH ROW EXECUTE FUNCTION touch_updated_at();
CREATE TRIGGER trg_deals_updated BEFORE UPDATE ON deals
  FOR EACH ROW EXECUTE FUNCTION touch_updated_at();

3c. Seed Data Generator

generate_seed.py
"""
Generates realistic CRM seed data and writes seed.sql.
Run: python generate_seed.py
Then: docker exec -i crm_pg psql -U crm -d crm < seed.sql
"""
import random, datetime

random.seed(42)  # Change for different but equally valid data

FIRST = ["Alice","Bob","Carol","David","Eva","Frank","Grace",
         "Henry","Iris","James","Karen","Leo","Maya","Nate",
         "Minh","Linh","Huy","Thanh","Bao","Khanh"]

LAST = ["Nguyen","Tran","Le","Smith","Jones","Brown","Wilson",
        "Patel","Chen","Kim","Park","Muller","Hoang","Pham"]

COMPANIES = [
    ("TechCorp VN","saas","Ho Chi Minh City","VN"),
    ("Acme Insurance","insurance","Sydney","AU"),
    ("Berlin Fintech GmbH","fintech","Berlin","DE"),
    ("SG Capital","fintech","Singapore","SG"),
    ("NY Ventures","fintech","New York","US"),
    ("Nordic Insurance AB","insurance","Stockholm","SE"),
    ("HCM Dev Studio","saas","Ho Chi Minh City","VN"),
    ("VietSoft","saas","Hanoi","VN"),
]

PRODUCTS = ["workers_comp","ctp","liability","property","cyber","life"]
OWNERS   = ["sales_alice","sales_bob","sales_carol","sales_david"]
SOURCES  = ["website","referral","cold_outreach","event","partner"]
STATUSES = ["active","active","active","lead","lead","inactive","churned"]
STAGES   = ["prospecting","qualified","proposal","negotiation",
            "closed_won","closed_won","closed_lost"]
STAGE_PROB = {"prospecting":10,"qualified":25,"proposal":50,
              "negotiation":75,"closed_won":100,"closed_lost":0}
TAG_POOL = ["vip","renewal-risk","upsell","new-logo","enterprise","smb"]

used_emails = set()

def unique_email(first, last, company):
    domain = company.lower().replace(" ","")[:12] + ".com"
    base   = f"{first.lower()}.{last.lower()}"
    email  = f"{base}@{domain}"
    if email in used_emails:
        email = f"{base}{random.randint(2,99)}@{domain}"
    used_emails.add(email)
    return email

def rand_date(start=730, end=1):
    d = random.randint(end, start)
    return (datetime.date.today() - datetime.timedelta(days=d)).isoformat()

def pg_arr(lst): return "NULL" if not lst else "ARRAY["+",".join(f'"{t}"' for t in lst)+"]"
def pg_s(s):    return "NULL" if s is None else "'"+s.replace("'","''")+"'"

NUM_CONTACTS, NUM_DEALS = 50, 30
contacts = []
for _ in range(NUM_CONTACTS):
    first, last, comp = random.choice(FIRST), random.choice(LAST), random.choice(COMPANIES)
    contacts.append({
        "name": f"{first} {last}", "email": unique_email(first,last,comp[0]),
        "company":comp[0], "industry":comp[1], "city":comp[2], "country":comp[3],
        "status":random.choice(STATUSES), "source":random.choice(SOURCES),
        "assigned_to":random.choice(OWNERS), "tags":random.sample(TAG_POOL,k=random.randint(0,3)),
        "notes":random.choice([None,"Demo requested","Budget approved","Referred by partner"]),
        "created_at":rand_date()
    })

cidx = list(range(NUM_CONTACTS)); random.shuffle(cidx)
deals = []
for i in range(NUM_DEALS):
    cid = cidx[i % NUM_CONTACTS]+1; stage = random.choice(STAGES)
    product = random.choice(PRODUCTS); company = contacts[cid-1]["company"]
    deals.append({
        "contact_id":cid, "stage":stage, "probability":STAGE_PROB[stage],
        "title":f"{company} – {product.replace('_',' ').title()} {random.choice(['licence','contract','renewal'])}",
        "value":round(random.uniform(5000,500000),-2), "owner":contacts[cid-1]["assigned_to"],
        "close_date":rand_date(60,-120), "product":product
    })

db.py — Database helpers

db.py
import os, psycopg2, psycopg2.extras
from dotenv import load_dotenv
load_dotenv()

def get_conn():
    return psycopg2.connect(
        host=os.getenv("PG_HOST"), port=os.getenv("PG_PORT"),
        dbname=os.getenv("PG_DB"), user=os.getenv("PG_USER"),
        password=os.getenv("PG_PASSWORD"),
        cursor_factory=psycopg2.extras.RealDictCursor
    )

def search_contacts(name=None, company=None, city=None,
                    country=None, industry=None, status=None,
                    assigned_to=None, limit=10):
    clauses, params = [], []
    if name:        clauses.append("name ILIKE %s");         params.append(f"%{name}%")
    if company:     clauses.append("company ILIKE %s");      params.append(f"%{company}%")
    if city:        clauses.append("city ILIKE %s");         params.append(f"%{city}%")
    if country:     clauses.append("country = %s");          params.append(country.upper())
    if industry:    clauses.append("industry = %s");         params.append(industry)
    if status:      clauses.append("status = %s");           params.append(status)
    if assigned_to: clauses.append("assigned_to = %s");      params.append(assigned_to)
    where = ("WHERE " + " AND ".join(clauses)) if clauses else ""
    sql = f"""SELECT id, name, email, company, industry, city,
                     country, status, assigned_to, tags, notes
              FROM contacts {where} ORDER BY name LIMIT %s"""
    params.append(limit)
    with get_conn() as conn:
        with conn.cursor() as cur:
            cur.execute(sql, params); return cur.fetchall()

def get_deals(stage=None, owner=None, min_value=None,
              max_value=None, product=None,
              closing_before=None, closing_after=None, limit=10):
    clauses, params = [], []
    if stage:          clauses.append("d.stage = %s");           params.append(stage)
    if owner:          clauses.append("d.owner = %s");           params.append(owner)
    if min_value:      clauses.append("d.value >= %s");          params.append(min_value)
    if max_value:      clauses.append("d.value <= %s");          params.append(max_value)
    if product:        clauses.append("d.product = %s");         params.append(product)
    if closing_before: clauses.append("d.close_date <= %s");     params.append(closing_before)
    if closing_after:  clauses.append("d.close_date >= %s");     params.append(closing_after)
    where = ("WHERE " + " AND ".join(clauses)) if clauses else ""
    sql = f"""SELECT d.id, d.title, d.value, d.stage, d.probability,
               d.owner, d.close_date, d.product,
               c.name AS contact_name, c.company, c.country
        FROM deals d JOIN contacts c ON c.id = d.contact_id
        {where} ORDER BY d.value DESC LIMIT %s"""
    params.append(limit)
    with get_conn() as conn:
        with conn.cursor() as cur:
            cur.execute(sql, params); return cur.fetchall()

def get_contact_deals(contact_id):
    sql = """SELECT d.*, c.name AS contact_name FROM deals d
             JOIN contacts c ON c.id = d.contact_id
             WHERE d.contact_id = %s ORDER BY d.created_at DESC"""
    with get_conn() as conn:
        with conn.cursor() as cur:
            cur.execute(sql, (contact_id,)); return cur.fetchall()

def get_pipeline_summary():
    sql = """SELECT stage, COUNT(*) AS deal_count,
             SUM(value) AS total_value, AVG(value) AS avg_value,
             SUM(value * probability / 100) AS weighted_value
             FROM deals GROUP BY stage ORDER BY total_value DESC"""
    with get_conn() as conn:
        with conn.cursor() as cur:
            cur.execute(sql); return cur.fetchall()

def get_contact_activities(contact_id, limit=10):
    sql = """SELECT a.type, a.summary, a.created_by, a.created_at,
                    d.title AS deal_title
             FROM activities a LEFT JOIN deals d ON d.id = a.deal_id
             WHERE a.contact_id = %s ORDER BY a.created_at DESC LIMIT %s"""
    with get_conn() as conn:
        with conn.cursor() as cur:
            cur.execute(sql, (contact_id, limit)); return cur.fetchall()

chatbot.py — Main chat loop

chatbot.py
import os, json
from openai import OpenAI
from rich.console import Console
from rich.markdown import Markdown
from dotenv import load_dotenv
import db, tools as tool_defs

load_dotenv()
console = Console()
client = OpenAI(base_url=os.getenv("OLLAMA_BASE_URL"), api_key="ollama")
MODEL  = os.getenv("OLLAMA_MODEL")

SYSTEM = """You are a helpful CRM assistant with access to tools that query
a PostgreSQL CRM database. Rules:
- Always use a tool to answer questions about contacts or deals — never guess.
- Summarise results clearly; use markdown tables for lists.
- Deal stages: prospecting → qualified → proposal → negotiation → closed_won / closed_lost.
- probability: prospecting=10%, qualified=25%, proposal=50%, negotiation=75%."""

TOOL_MAP = {
    "search_contacts":       db.search_contacts,
    "get_deals":             db.get_deals,
    "get_contact_deals":     db.get_contact_deals,
    "get_pipeline_summary":  db.get_pipeline_summary,
    "get_contact_activities":db.get_contact_activities,
}
MAX_TOOL_CALLS = 5  # guard against infinite tool loops

def run_tool(name, args):
    fn = TOOL_MAP.get(name)
    if not fn: return {"error": f"Unknown tool: {name}"}
    try:    return [dict(r) for r in fn(**args)]
    except Exception as e: return {"error": str(e)}

def chat(history):
    n = 0
    while True:
        resp = client.chat.completions.create(
            model=MODEL, messages=history,
            tools=tool_defs.TOOLS, tool_choice="auto")
        msg = resp.choices[0].message
        if msg.tool_calls and n < MAX_TOOL_CALLS:
            history.append(msg)
            for tc in msg.tool_calls:
                args = json.loads(tc.function.arguments)
                console.print(f"[dim]  → calling {tc.function.name}({args})[/dim]")
                history.append({"role":"tool","tool_call_id":tc.id,
                                 "content":json.dumps(run_tool(tc.function.name,args),default=str)})
                n += 1
        else:
            history.append({"role":"assistant","content":msg.content})
            console.print(Markdown(msg.content or "")); return history

def main():
    console.print("[bold blue]CRM Chatbot — P1 (Tool Calling)[/bold blue]")
    console.print(f"[dim]Model:[/dim] {MODEL}\n")
    history = [{"role":"system","content":SYSTEM}]
    while True:
        try:    user_input = console.input("[bold]You:[/bold] ").strip()
        except: break
        if user_input.lower() in ("exit","quit","bye"): break
        if not user_input: continue
        history.append({"role":"user","content":user_input})
        history = chat(history); print()

if __name__ == "__main__": main()

Hardening Checklist

  • Add a read-only Postgres role: CREATE USER crm_readonly WITH PASSWORD '...'; GRANT SELECT ON ALL TABLES IN SCHEMA public TO crm_readonly; — update .env to use it
  • MAX_TOOL_CALLS = 5 guard is already in chatbot.py — prevents infinite tool loops
  • run_tool() has try/except — errors return a message back to the LLM rather than crashing
  • Consider trimming history after ~20 turns to avoid context window overflow on long sessions

Phase 2 — pgvector

Adds semantic search for fuzzy queries P1 can't handle

  • "Find contacts similar to our Singapore clients"
  • "Anyone in fintech we haven't contacted recently"
  • "Find companies that look like Acme Insurance"

P2 adds: pgvector extension on the same Postgres instance · contact_embeddings table with HNSW index (vector(768)) · local embedding model (nomic-embed-text via Ollama) · background sync worker · hybrid queries: embedding <=> $1 + SQL WHERE filters in one round-trip.

The tool calling layer built in P1 is unchanged — P2 just adds a new semantic_search_contacts tool alongside the existing ones.
